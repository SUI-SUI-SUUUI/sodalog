# そだログ

そだログは、LINEから園芸作業の記録を送るだけで、GoogleスプレッドシートとGoogle
Driveに自動整理する園芸記録管理アシスタントです。

## 概要

庭仕事中にパソコンを開かなくても、スマホのLINEから作業内容を送るだけで、日付・場所・植物名・作業内容を記録できます。

番号を選びながら入力する方法と、1行テキストで送る方法の両方に対応しています。記録後に画像を送ると、Google
Driveへ保存し、スプレッドシートの該当行へURLを追記します。

## 解決したい課題

園芸作業の記録は、写真・日付・場所・植物名・作業内容が分散しやすく、あとから振り返りにくいという課題があります。

そだログでは、普段使っているLINEを入口にすることで、庭仕事の流れを止めずに記録を残せる仕組みを作ります。

## 入力方法

### 番号選択で記録する

「記録する」と送ると、作業日 → 育成拠点 → 場所 → 詳細場所 → 植物名 →
作業内容 →
メモの順で入力できます。入力途中は「中止」でキャンセルできます。

### 1行テキストで記録する

次の形式でも記録できます。

```text
2026/07/13_北側_中央_つつじ_剪定
```

形式：`作業日_場所_詳細場所_植物名_作業内容`

### その他のコマンド

送信テキスト 動作

---

記録する / 記録 / 園芸記録 番号選択による記録を開始
最近の記録 / 最近 / 履歴 直近の記録を表示
使い方 / ヘルプ / メニュー 操作方法を表示
中止 / キャンセル 入力途中の記録を中止

## 現在の利用範囲とセキュリティ方針

現在のそだログは、EDD2026に向けたMVPとして、開発者本人のみが利用する構成です。

LINEから届いたイベントの送信元ユーザーIDを確認し、Google Apps
Scriptのスクリプトプロパティに登録した本人のLINEユーザーIDと一致する場合だけ、メッセージ処理・画像保存・スプレッドシートへの記録を行います。

```text
LINEからWebhookを受信
  ↓
送信元のLINEユーザーIDを確認
  ├─ 許可された本人 → 記録処理を実行
  └─ 本人以外       → 処理せず終了
```

許可するユーザーIDは、次のスクリプトプロパティで管理しています。

```text
ALLOWED_LINE_USER_ID
```

アクセストークン、チャンネルシークレット、保存先IDなども、コードへ直接記載せず、Google
Apps Scriptのスクリプトプロパティで管理しています。

### MVPでの対応範囲

- 開発者本人だけが利用
- 許可されたLINEユーザーIDだけ処理
- 未許可ユーザーからのイベントは保存・返信しない
- 秘密情報や保存先IDはスクリプトプロパティで管理
- LINE公式アカウントの追加用URLやQRコードは一般公開しない

### 一般公開時に必要な対応

現在のGoogle Apps Script単体構成では、LINEから送られる
`X-Line-Signature` を公式に案内された方法で検証することが困難です。

一般公開する場合は、Cloud RunなどをLINE
Webhookの受付口として追加し、署名検証を完了したイベントだけをバックエンドへ渡す構成を検討します。

```text
LINE
  ↓
Cloud Runなど
  ↓ X-Line-Signatureを検証
Google Apps Script
  ↓
Google Spreadsheet / Google Drive
```

この対応は、現在の本人限定MVPには導入せず、一般公開版へ移行する段階で実施します。

## コード構成

Google Apps Script のコードは `gas/` 配下で機能別に分割しています。

---

ファイル 役割

---

`00_Config.gs` 定数・シート名・ヘッダー定義

`01_Webhook.gs` Webhook受信（`doPost` /
`doGet`）、テキスト・画像の振り分け

`02_RecordFlow.gs` 番号選択による園芸記録入力フロー

`03_ValidationDate.gs` 1行テキストの検証・日付処理

`04_Image.gs` 画像保存・Driveフォルダ管理

`05_HelpHistory.gs` 使い方・最近の記録

`06_SheetsSessions.gs` スプレッドシート操作・ユーザーセッション管理

`07_LineUtilities.gs` LINE返信・JSON応答・デバッグログ

`08_RichMenu.gs` リッチメニューの作成・設定

`99_Tests.gs` 動作確認用テスト

---
